tearDown即便test方法抛出异常也会执行，但前提是setUp方法执行成功。

setUpClass()一个类执行一次，需以classmethod装饰器包装。tearDownClass也是如此。

TestCase是最小的可测试单位，用以从逻辑上将一系统相关的测试案例组织到一起。每一个案例运行中都会新建一个类实例（每个实例只运行一个测试方法：因而setUp/TearDown方法也会重新执行一次）。

unittest.main()

或

suite = unittest.TestSuite()
suite.addTest(MyTestCase('test_method_1')
suite.addTest(MyTestCase('test_method_2')

都是如此。

验证：

import unittest

class MyTest(unittest.TestCase):
    def setUp(self):
        print('')
        print(id(self))

    def test_id1(self):
        pass

    def test_id2(self):
        pass


if __name__ == '__main__':
    unittest.main()
